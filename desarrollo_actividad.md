## Instrucción general de la actividad:

La unidad de Pabellón Quirúrgico del Hospital regional ha solicitado a la unidad de TI de la institución generar una base de datos para poder guardar el registro de las intervenciones Quirúrgicas realizadas en dicha unidad y con esto poder cumplir con los informes de protocolo quirúrgico solicitados por el MINSAL.

Para cumplir con este requerimiento se necesita generar un script en lenguaje sql que contemple la creación de una base de datos MySql con sus respectivas tablas, registros y consultas planteados en las siguientes 3 preguntas, las cuales deben ser subidas con sus respectivas respuestas a la plataforma.

1. Se solicita generar script de una base de daros de nombre “BD_IQ” que contenga las presentes tablas y campos para poder almacenar la información de las actividades realizadas en pacientes en la unidad de pabellón.

```sql
CREATE DATABASE IF NOT EXISTS BD_IQ;

USE BD_IQ;

CREATE TABLE previsiones(
  prev_id INT UNSIGNED PRIMARY KEY,
  prev_glosa VARCHAR(200) NOT NULL UNIQUE
);

CREATE TABLE comunas(
  comu_id INT UNSIGNED PRIMARY KEY,
  comu_glosa VARCHAR(200) NOT NULL UNIQUE
);

CREATE TABLE especialidades(
  esp_id INT UNSIGNED PRIMARY KEY,
  esp_glosa VARCHAR(1000) NOT NULL
);

CREATE TABLE prestaciones(
  prest_id INT UNSIGNED PRIMARY KEY,
  prest_fonasa CHAR(7) NOT NULL,
  prest_glosa VARCHAR(2500) NOT NULL,
  prest_valor DECIMAL(12,2) NOT NULL
);

CREATE TABLE pacientes(
  pac_id INT UNSIGNED PRIMARY KEY,
  pac_nombres VARCHAR(500) NOT NULL,
  pac_apellido_paterno VARCHAR(500) NOT NULL,
  pac_apellido_materno VARCHAR(500) NOT NULL,
  pac_edad INT,
  prev_id INT,
  comu_id INT UNSIGNED,
  # existe un error con la cantidad de registros de prev_id con la tabla previsiones  
  # FOREIGN KEY(prev_id) REFERENCES previsiones(prev_id),
  FOREIGN KEY(comu_id) REFERENCES comunas(comu_id)
);

CREATE TABLE medicos(
  med_id INT UNSIGNED PRIMARY KEY,
  med_nombres VARCHAR(500) NOT NULL,
  med_apellido_paterno VARCHAR(500) NOT NULL,
  med_apellido_materno VARCHAR(500) NOT NULL,
  med_edad INT,
  esp_id INT UNSIGNED,
  FOREIGN KEY(esp_id) REFERENCES especialidades(esp_id)
);

CREATE TABLE actos_quirurgicos(
  acto_id INT UNSIGNED PRIMARY KEY,
  acto_folio VARCHAR(500) NOT NULL,
  acto_fecha VARCHAR(500) NOT NULL,
  med_id INT UNSIGNED,
  prest_id INT UNSIGNED,
  pac_id INT UNSIGNED,
  FOREIGN KEY(med_id) REFERENCES medicos(med_id),
  FOREIGN KEY(prest_id) REFERENCES prestaciones(prest_id),
  FOREIGN KEY(pac_id) REFERENCES pacientes(pac_id)
);
```

3. Dado que ya se cuenta con información en la base de datos de la unidad de Pabellones Quirúrgicos, se necesita hacer uso de este recurso para la toma correcta de decisiones, por lo cual se solicita pueda dar respuesta a las siguientes solicitudes mediante consultas SQL a la base de datos.

- Cantidad de Prestaciones realizadas.

  - Prest_Id
  - Prest_Fonasa
  - Prest_Glosa
  - Cantidad

```sql
SELECT a.prest_id, p.prest_fonasa, p.prest_glosa, COUNT(*) AS cantidad
FROM actos_quirurgicos a
LEFT JOIN prestaciones p ON a.prest_id = p.prest_id
GROUP BY a.prest_id ORDER BY cantidad DESC;
```

- Total, valor de prestaciones entre las fechas 01/03/2021 al 31/05/2021 por Especialidad médica.

  - Esp_Id
  - Esp_Glosa
  - Total

```sql
SELECT e.esp_id, e.esp_glosa, SUM(p.prest_valor) AS total
FROM actos_quirurgicos a
INNER JOIN medicos m ON med.med_id = a.med_id
INNER JOIN especialidades e ON e.esp_id = m.esp_id
INNER JOIN prestaciones p ON p.prest_id = a.prest_id
WHERE a.acto_fecha BETWEEN '2021-03-01' AND '2021-05-31'
GROUP BY e.esp_id;
```

- Promedio de Actos Quirúrgicos por profesional médico.

  - Med_Id
  - Med_Nombres
  - Med_Apellido_Pat
  - Med_Apelliedo_Mat
  - Promedio
 
```sql
SELECT m.med_id, m.med_nombres, m.med_apellido_paterno, m.med_apellido_materno, COUNT(*) AS promedio
FROM actos_quirurgicos a
INNER JOIN medicos m ON a.med_id = m.med_id
```
